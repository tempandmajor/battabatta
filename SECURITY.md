# Security Policy

Report vulnerabilities privately to OMS2 maintainers. Do not open public issues for security-sensitive reports.

Include:

- Affected route, API, policy, or workflow.
- Reproduction steps.
- Potential impact.
- Suggested fix if available.

High priority areas:

- Supabase RLS bypasses.
- Private profile or exact location exposure.
- Stripe webhook spoofing or payment state confusion.
- Offer/message access across blocked or unrelated users.
- Account deletion/export defects.
